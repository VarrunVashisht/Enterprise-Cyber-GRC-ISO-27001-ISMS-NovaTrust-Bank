# NovaTrust Bank
## Cybersecurity Governance Program
### Module 09 – Cyber Risk Treatment Plan

---

# 1. Introduction

Cyber risk treatment is the process of selecting and implementing controls to reduce cybersecurity risks to an acceptable level.

Following the enterprise cyber risk assessment conducted in Module 07, NovaTrust Bank has identified several critical and high-level cyber risks that require mitigation.

The purpose of this risk treatment plan is to:

• define risk mitigation strategies  
• identify security controls required to reduce cyber risk  
• assign ownership of risk mitigation activities  
• prioritize cybersecurity investments

This plan aligns with ISO/IEC 27001 risk treatment requirements and the NIST Cybersecurity Framework.

---

# 2. Risk Treatment Options

NovaTrust Bank uses four primary risk treatment strategies.

## Risk Mitigation

Implementing security controls to reduce the likelihood or impact of a risk.

## Risk Transfer

Transferring risk to third parties such as through insurance or outsourcing.

## Risk Acceptance

Accepting the risk when mitigation costs exceed potential impact.

## Risk Avoidance

Eliminating activities that create unacceptable risk.

Most cyber risks within NovaTrust Bank are treated through mitigation using strong security controls.

---

# 3. Risk Treatment Strategy

The cyber risk treatment strategy focuses on implementing layered security controls across the following domains:

• identity and access management  
• network security  
• endpoint protection  
• application security  
• cloud security  
• fraud detection  
• third-party risk management

These security domains collectively strengthen the bank’s cyber resilience.

---

# 4. Cyber Risk Treatment Plan

The following table outlines the primary cyber risks identified in Module 07 and the mitigation strategies implemented to reduce those risks.

| Risk ID | Risk Description | Treatment Strategy | Responsible Team |
|-------|------------------|-------------------|----------------|
| CR-01 | Ransomware attack on core banking infrastructure | Deploy advanced endpoint detection and response (EDR), network segmentation, secure backups | Security Operations |
| CR-02 | Phishing leading to credential compromise | Implement multi-factor authentication (MFA) and phishing awareness training | Identity & Security Team |
| CR-03 | Cloud misconfiguration exposing customer data | Deploy cloud security posture management (CSPM) tools and automated configuration monitoring | Cloud Security Team |
| CR-04 | Insider misuse of privileged access | Implement privileged access management (PAM) and activity monitoring | Identity Security Team |
| CR-05 | Third-party vendor compromise | Establish third-party risk management program and vendor security assessments | Risk & Compliance Team |
| CR-06 | DDoS attack disrupting online banking | Deploy DDoS protection services and traffic filtering mechanisms | Network Security Team |
| CR-07 | Payment fraud through compromised accounts | Implement advanced fraud detection analytics and transaction monitoring | Fraud Prevention Team |
| CR-08 | API security vulnerabilities | Implement API security testing and secure development practices | Application Security Team |
| CR-09 | Malware infection across employee devices | Deploy endpoint protection platforms and vulnerability management | Security Operations |
| CR-10 | Data leakage through misconfigured storage | Implement data loss prevention (DLP) and encryption controls | Data Security Team |

# 5. Implementation Priorities

Cybersecurity initiatives are prioritized based on risk severity.

## Priority 1 – Critical Risks

Immediate mitigation efforts focus on:

• ransomware defense capabilities  
• identity protection and multi-factor authentication  
• fraud prevention systems  
• protection of core banking infrastructure

## Priority 2 – High Risks

Secondary mitigation efforts include:

• strengthening cloud security monitoring  
• enhancing API security  
• improving endpoint security and malware protection

# 8. Risk Ownership and Accountability

Each cyber risk identified in the risk register is assigned an owner responsible for implementing mitigation strategies.

Risk owners are responsible for:

• implementing required security controls  
• monitoring risk mitigation progress  
• reporting risk status to the cybersecurity governance team

This accountability ensures effective management of cyber risk across the organization.

---

# 9. Monitoring Risk Treatment Progress

NovaTrust Bank monitors the effectiveness of risk mitigation through:

• security performance metrics  
• vulnerability assessments  
• internal security audits  
• incident response analysis

Progress reports are provided regularly to executive leadership and the Board Risk Committee.

NovaTrust Bank periodically reviews its risk treatment plan to:
